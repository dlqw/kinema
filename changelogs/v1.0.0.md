# AniMaker v1.0.0 Release Notes

Release Date: March 19, 2026

---

## Overview

AniMaker v1.0.0 is the first stable release of the TypeScript-based animation rendering framework. This release provides a complete foundation for creating 2D animations with a focus on type safety, composability, and ease of use.

---

## What's New

### Core Framework

**Type-Safe Foundation**
- Complete TypeScript type system with branded types for compile-time safety
- Immutable data patterns throughout the framework
- Comprehensive JSDoc documentation for all public APIs

**Render Objects**
- Abstract `RenderObject` base class with state management
- `VectorObject` for shapes (circle, rectangle, polygon, line, arc, path)
- `TextObject` for text rendering with font configuration
- `GroupObject` for hierarchical object composition

**Scene Management**
- `Scene` class for organizing and managing animated objects
- Scene scheduling and time management
- Scene snapshots for state preservation
- Background configuration support

### Animation System

**Animation Types**
- Base `Animation` class with `AnimationBuilder` for fluent API
- `TransformAnimation` for generic transform interpolation
- `FadeAnimation` (FadeIn, FadeOut, FadeTo)
- `RotateAnimation` for rotation animations
- `MoveAnimation` for linear and path-based movement
- `AnimationGroup` for parallel, sequence, and lagged composition

**Easing Functions**
- 20+ built-in easing functions
- Categories: Linear, Smooth, Quad, Cubic, Quart, Quint, Sine, Expo, Circ, Elastic, Back, Bounce
- Custom easing function support

### Timeline Control

**Timeline Features**
- Play/pause/stop controls
- Forward and reverse playback direction
- Speed control from 0.1x to 10x
- Duration and progress tracking
- Loop support

**Keyframes**
- `KeyframeTrack<T>` for property animation
- `KeyframeManager` for multi-property management
- Four interpolation types: Linear, Step, Smooth, Cubic

**Time Markers**
- Four marker types: Section, Event, Label, Beat
- `MarkerCollection` for marker management
- `MarkerBuilder` for fluent marker creation

### Event System

**EventEmitter**
- Type-safe event handling with generics
- Event bubbling through parent chain
- Priority-based listener execution
- One-time listeners
- Unsubscribe mechanism

**Event Types**
- **Animation Events**: start, update, complete, repeat, pause, resume, cancel, error
- **Scene Events**: init, objectAdded, objectRemoved, objectChanged, frameRender, beforeFrameRender, afterFrameRender, update, pause, resume, clear
- **Render Events**: start, beforeRender, afterRender, error, warning, performance, resize, contextLoss, contextRestored

**EventBus**
- Global singleton for framework-wide event communication
- Channel-based organization (Animation, Scene, Render, Input, Timeline, Export, System)
- Event history tracking for debugging

### Factory Functions & High-Level API

**Shape Factories**
```typescript
Circle(radius, color?)
Rectangle(width, height, color?)
Polygon(points, color?)
Line(start, end, color?)
Arc(radius, startAngle, endAngle, color?)
Path(commands, color?)
```

**Animation Factories**
```typescript
fade(duration, options?)
move(path, duration, options?)
rotate(angle, duration, options?)
scale(factor, duration, options?)
parallel(animations)
sequence(animations)
lagged(animations, delay)
```

**Scene Factories**
```typescript
createScene(width, height, config?)
createHDScene(config?)
animateSequentially(objects, animations)
```

**Chain API**
```typescript
Chain.circle(100, '#ff0000')
  .position(50, 50)
  .rotate(45)
  .scale(1.5)
  .opacity(0.8)
```

### Export System

**Supported Formats**
- Image sequences: PNG, JPEG, WebP
- Animated GIF with configurable quality
- Video: WebM (VP8/VP9), MP4 (H.264)

**Export Features**
- Frame-by-frame encoding
- Quality controls for each format
- Progress tracking during export
- Canvas-based rendering abstraction

## Migration Guide

If you're upgrading from a pre-release version:

1. **Update your dependencies**:
   ```bash
   npm install animaker@1.0.0
   ```

2. **Update imports** - The package structure is now stable:
   ```typescript
   // Core types
   import { ObjectId, Time, Alpha } from '@animaker/core/types';

   // Render objects
   import { Circle, Rectangle } from '@animaker/core/factory';

   // Animations
   import { fade, move, parallel } from '@animaker/core/factory';

   // Events
   import { EventEmitter, EventBus } from '@animaker/core/events';
   ```

3. **Review type changes** - Branded types now provide better type safety:
   ```typescript
   // Before (loose typing)
   const id: string = 'obj-123';

   // After (branded type)
   const id: ObjectId = ObjectId.create('obj-123');
   ```
